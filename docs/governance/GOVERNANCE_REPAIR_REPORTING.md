# Governance Repair Reporting

Phase 3V reframes LAIF external-framework reporting as governance repair assessment, not form-fit compliance grading. LAIF is a governance measurement and systemic failure detection engine: it asks whether a document exposes, closes, or leaves open operational failure pathways.

## Doctrine

External-framework mode does not require source documents to imitate LAIF-native structure or terminology. The front-facing report evaluates governance repair adequacy and operational control closure, including:

- systemic governance repair value;
- operational closure;
- evidence sufficiency;
- accountability closure;
- lifecycle control;
- residual-risk closure;
- implementation readiness; and
- failure-pathway risk.

Formal LAIF-native certification remains available for LAIF-native documents. For external instruments, LAIF-native construct coverage is an internal diagnostic shown only under the technical appendix / internal diagnostic boundary.

## External-framework report answer set

The report is designed to help a government, regulator, procurement team, or assurance reviewer answer:

1. What kind of governance document is this?
2. What institutional force does it have?
3. What does it actually control?
4. What does it only appear to control?
5. Where could systemic governance failure still occur?
6. What operational controls are required?
7. What should the reviewer do next?

## Deterministic field basis

Governance repair fields are derived from existing diagnostics: auditability score, enforceability score, conceptual proximity, structural score, evidence trace count, remediation patch count/severity, formal compliance flags, sector profile signals, and detected control gaps. The mapping is deterministic and does not alter scoring weights, formal LAIF-native validation, or `validate.py` behavior.

## Technical appendix boundary

External-framework reports use this certification wording:

> LAIF-native certification: Not claimed / not applicable to this external-framework assessment.

The appendix may still show LAIF-native construct coverage so reviewers can understand which LAIF-native controls would need repair if the document were converted into a LAIF-native instrument.
